The differences of looping

1/24/2016

This week at DBC, we have transitioned from Ruby into JavaScript. It has been a rough week of learning new syntax. The lack of help when it comes to errors is also something I have had to learn to miss from Ruby. Another big difference I have come to realizee is the power of built in methods that Ruby has. Some of these built in methods contain large amounts of code that condense the amount of code that you need to type.

Looping in JavaScript is similar to using the while method in Ruby. In the above example, we use the "for" method to iterate over array using the variable index. We set the index to start at 0, or the first index of an array, and continue until index reaches one number below array.length or the length of the array. The index++ will increase the number of index by 1 each time so it will execute the block for each element of the array. The block puts the element into the console.

In Ruby, we can use the each method to iterate over every element of the array and the syntax is much simpler. As you can see the code is much easier to read. For the given array, we run the block {|x| x + 1} which takes each element as x and adds 1 to it. The power and simplicity that Ruby brings is amazing and I am glad I was able to temper my logic skills using Ruby before I dove into other languages.
